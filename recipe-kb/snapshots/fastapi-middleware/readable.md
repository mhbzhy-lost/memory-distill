- [fastapi-middleware-1] Middleware ¶

- [fastapi-middleware-2] You can add middleware to FastAPI applications.

- [fastapi-middleware-3] A "middleware" is a function that works with every request before it is processed by any specific path operation . And also with every response before returning it.

- [fastapi-middleware-4] It takes each request that comes to your application.

- [fastapi-middleware-5] It can then do something to that request or run any needed code.

- [fastapi-middleware-6] Then it passes the request to be processed by the rest of the application (by some path operation ).

- [fastapi-middleware-7] It then takes the response generated by the application (by some path operation ).

- [fastapi-middleware-8] It can do something to that response or run any needed code.

- [fastapi-middleware-9] Then it returns the response .

- [fastapi-middleware-10] Technical Details

- [fastapi-middleware-11] If you have dependencies with yield , the exit code will run after the middleware.

- [fastapi-middleware-12] If there were any background tasks (covered in the Background Tasks section, you will see it later), they will run after all the middleware.

- [fastapi-middleware-13] Create a middleware ¶

- [fastapi-middleware-14] To create a middleware you use the decorator @app.middleware("http") on top of a function.

- [fastapi-middleware-15] The middleware function receives:

- [fastapi-middleware-16] The request .

- [fastapi-middleware-17] A function call_next that will receive the request as a parameter. This function will pass the request to the corresponding path operation . Then it returns the response generated by the corresponding path operation .

- [fastapi-middleware-18] This function will pass the request to the corresponding path operation .

- [fastapi-middleware-19] Then it returns the response generated by the corresponding path operation .

- [fastapi-middleware-20] You can then further modify the response before returning it.

- [fastapi-middleware-21] import time from fastapi import FastAPI , Request app = FastAPI () @app . middleware ( "http" ) async def add_process_time_header ( request : Request , call_next ): start_time = time . perf_counter () response = await call_next ( request ) process_time = time . perf_counter () - start_time response . headers [ "X-Process-Time" ] = str ( process_time ) return response

- [fastapi-middleware-34] Multiple middleware execution order ¶

- [fastapi-middleware-35] When you add multiple middlewares using either @app.middleware() decorator or app.add_middleware() method, each new middleware wraps the application, forming a stack. The last middleware added is the outermost , and the first is the innermost .

- [fastapi-middleware-36] On the request path, the outermost middleware runs first.

- [fastapi-middleware-37] On the response path, it runs last.

- [fastapi-middleware-38] For example:

- [fastapi-middleware-39] app . add_middleware ( MiddlewareA ) app . add_middleware ( MiddlewareB )

- [fastapi-middleware-40] This results in the following execution order:

- [fastapi-middleware-41] Request : MiddlewareB → MiddlewareA → route

- [fastapi-middleware-42] Response : route → MiddlewareA → MiddlewareB

- [fastapi-middleware-43] This stacking behavior ensures that middlewares are executed in a predictable and controllable order.
